# Task Notes

Use `.agents/tasks/todo-(name).md` for larger or multi-step work when a persistent task note is useful.

This is optional. Do not create a task note for every small edit.

Recommended structure:

```md
# Task Name

## Objective

Short statement of the outcome.

## Checklist

- [ ] Read the relevant code and tests
- [ ] Implement the change
- [ ] Run verification
- [ ] Summarize risks or follow-ups

## Verification

- Commands run
- Manual checks performed

## Review Notes

- What changed
- Any remaining risk or follow-up
```

Suggested filename example:

```text
.agents/tasks/todo-admin-render-preview.md
```
