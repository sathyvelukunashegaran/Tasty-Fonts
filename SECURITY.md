# Security Policy

## Reporting A Vulnerability

Please report security vulnerabilities privately through GitHub's private security advisory flow for this repository:

- [Report a vulnerability](https://github.com/sathyvelukunashegaran/Tasty-Custom-Fonts/security/advisories/new)

Do not open public issues or pull requests for suspected vulnerabilities.

## In Scope

Security reports are in scope when they affect Tasty Custom Fonts itself, including:

- frontend runtime behavior
- admin screens and actions
- REST API endpoints
- upload, import, storage, and generated CSS handling
- update and release delivery logic that ships with the plugin

## Out Of Scope

The following are generally out of scope unless the plugin introduces the vulnerability directly:

- third-party font provider infrastructure or service availability
- vulnerabilities that exist only in Google Fonts, Bunny Fonts, Adobe Fonts, or WordPress core
- support requests, configuration mistakes, or non-security feature gaps

## Response Expectations

- Initial acknowledgment: within 5 business days
- Follow-up status updates: at least weekly while the report is active
- Coordinated disclosure: after a fix or mitigation is available, when feasible
